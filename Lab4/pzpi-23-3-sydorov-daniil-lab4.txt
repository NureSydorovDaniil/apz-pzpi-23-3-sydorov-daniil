2 ЗАВДАННЯ
В цій лабораторній роботі необхідно показати як можна масштабувати бекенд системи для роботи із великим навантаженням.

3 ОПИС ВИКОНОЇ РОБОТИ
Під час виконання цієї роботи було реалізовано масштабовану архітектуру Web-застосунку. Основна мета роботи полягала у демонструванні можливості масштабування бекенд частини системи для обробки великої кількості запитів.
На першому етапі було розроблено простий бекенд сервер на основі платформи Node.js із використанням фреймворку Express. Сервер забезпечує обробку HTTPS запитів, створення, отримання та видалення замовлень.
Було реалізовано адміністративну панель, яка дозволяє переглядати всі замовлення та видаляти їх при потребі. 
Для демонстрації масштабування застосунку було використано відокремлення клієнтської та серверної частини. Бекенд може бути запущений як окремий сервіс. Система підтримує базову архітектуру, яка дозволяє масштабування бекенд рівня без заміни клієнтської частини.
У роботі було використано стратегію горизонтального масштабування бекенд частини, основна ідея полягає у можливості запуску декількох інстант сервера, які можуть паралельно обробляти запити.
Масштабування системи забезпечується завдяки:
•	Використанню архітектури клієнт-серверу з чітким розподіленням фронтнеду та бекенду
•	Реалізація REST API на базі Node.js
•	Використання stateless підходу для обробки https запитів
•	Можливість розгортання кількох копій сервера

У даній системі також виділено декілька вузьких місць, а саме файлове зберігання даних, яке реалізовано через json. При великій кількості запитів можливі конфлікти запису. Один сервісний інстанс без масштабування всі запити обробляються через Node.js. Також відсутнє кешування, кожен запит звертається до джерела напряму.
Для перевірки навантаження було проведено навантажувальне тестування, яке імітує одночасні http запити до бекенд сервера. Тести включали в себе надсилання великої кількості Get-запитів, імітацію створення замовлень, перевірку стабільності системи.
