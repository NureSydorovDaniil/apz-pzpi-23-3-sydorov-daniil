3 ОПИС ВИКОНОЇ РОБОТИ
Патерн – це типовий спосіб вирішення певної проблеми, що часто зустрічається при проектуванні архітектури програм.
Використання патернів дозволяє зробити код більш зрозумілішим, гнучким і придатним для подальшого розширення.

Згідно з класифікацією GoF, шаблони проєктування поділяються на три основні групи:
•	Породжувальні
•	Структууровані
•	Поведінкові

3.2 Опис обраного патерну
Для доповіді було обрано патерн Factory Method (Фабричний метод)
Патерн долежить до групи породжувальних патернів.
Використвується для гнучкого створення об'єктів у програмі,  допомагає відмовитись від створення об'єктів за допомогою оператора new, замінивши його викликом особливого фабричного методу.
Використовується для:
•	Типів і залежностей об’єктів, з якими повинен працювати код, невідомі зазделегіть.
•	Коли є можливість надати користувачам можливість розширювати частини фреймворку та бібліотеки
•	Коли є бажання зекономіти системні рессурси, повторно використовуючи вже створені об’єкти, замість породження нових.
Переваги:
•	Гнучкість 
•	Легке розширення системи
•	Менша залежність між класами
•	Відповідність принципу Open/closed
•	Краща підтримка коду

Недоліки:
•	Збільшення кількості класів
•	Складніша структура проєкту 
•	Зайвий для невеликих програм
•	Потребує бульше часу на проєктування

Програмний код наведений на мові програмування Java
1.	Приклад використання без Factory Method

class Truck {
	void deliver() {
		System.out.println(«Доставка вантажівкою»);
	}
}

public class Main {
	public static void main(String[] args) {
		Truck truck – new Truck();
		truck.deliver();
	}
}

2.	Приклад використання з Factory Method

interface Transport {
	void deliver();
}

class Truck implements Transport {
	public void deliver() {
		System.out.println(«Доставка вантажівкою»);
	}
}

abstract class Logistics {
	abstract Transport createTransport();
}

class RoadLogistics extends Logistics {
	Transport createTransport() {
		return new Truck();
	}
}
